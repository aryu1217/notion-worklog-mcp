# Current Work Documentation Template

Write the document in the section order below.

## Purpose
- Summarize what this work is trying to achieve.

## Changes
- Summarize the behavior, UI, or structure that changed.

## Important Files
- List the most relevant files and what each one is responsible for.

## Technical Notes
- Record implementation decisions, tradeoffs, or troubleshooting notes.

## Remaining Work
- List follow-up work, open questions, or risks that still remain.
